Phase 2 — Submit Claim Onchain
ClaimShield dApp Bundles Claim Data
The dApp securely binds the FHIR Claim ID and the World ID ZK Proof into a payload. It encrypts this payload so it can't be read publicly.

Smart Contract Emits Claim Event Real TX
The blockchain records the claim submission event. The medical data is completely opaque and protected on-chain.
Phase 3 — Chainlink CRE / TEE Computation
Chainlink Nodes Detect Claim
The decentralized oracle network picks up the on-chain event securely routing the encrypted payload to a confidential computing enclave.
🔒 Inside Chainlink CRE TEE · All data stays here
Step 1 — Decrypt Payload
Payload is securely decrypted inside the TEE using the enclave's private keys. The FHIR Claim ID and World ID proof are accessed privately.
Step 2a — Verify World ID Real API
Directly queries World ID endpoint from the enclave. Ensures claim isn't a duplicate and claimant is a verified human.
Step 2b — Fetch Electronic Health Record Real API
Directly queries the hospital's FHIR API to fetch the actual diagnosis (e.g., ICD-10 code J06.9), treatment date, and billed cost.
Step 3 — Eligibility Logic
Checks ICD-10 code against covered conditions list · Validates treatment date within coverageStart–coverageEnd · Calculates payout at 80% of billed, capped at $500. All logic runs in WASM. Never exposed.
ICD-10 code diagnosis text billed amount treatment date
Step 4 — Finalize Verdict
A non-reversible compliance hash is generated. Only the final output (Approved/Denied, Payout) leaves the TEE. The medical data is securely destroyed.
Phase 4 — Write Verdict Onchain
Chainlink CRE writes directly on-chain
Policy Registry Updated Real TX
The Chainlink enclave signs and submits the final status (e.g. APPROVED) and computed payout to the blockchain.
✅ APPROVED → USDC Payout
Smart contract automatically transfers USDC from the settlement liquidity pool directly to the user's wallet.
❌ DENIED → Audit Trail Updated
Smart contract logs the denial reason code (e.g. NOT_COVERED) without ever exposing the user's diagnosis on-chain.
User interfaces updates
Phase 5 — Claim Settled & Available for Public Verification
The claimant successfully receives reimbursement. Auditors can verify the Compliance Hash and enclave signature, while medical records remain strictly private.
